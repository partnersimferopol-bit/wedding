# Заявки в админку + уведомления в сообщения сообщества ВК

> **Подробная пошаговая инструкция:** [INSTRUKCIYA-ZAYAVKI-VK.md](INSTRUKCIYA-ZAYAVKI-VK.md)

Игра отправляет заявку на **Cloudflare Worker**, он:
1. сохраняет заявку в хранилище;
2. шлёт текст в **сообщения ВК** (от имени вашего сообщества).

---

## Шаг 1. Сообщество ВК — ключ и куда слать

1. Откройте сообщество → **Управление** → **Работа с API** → **Создать ключ**.
2. Права: **Сообщения сообщества** (и при необходимости «Управление сообществом»).
3. Скопируйте **ключ доступа сообщества** → это `VK_GROUP_TOKEN`.

### Куда придёт уведомление (`VK_NOTIFY_PEER_ID`)

Сообщество **не может** «написать само себе» во входящие. Обычно делают так:

**Вариант А — беседа администраторов (рекомендуется)**  
1. В сообществе: **Сообщения** → **Создать беседу** → добавьте себя и менеджеров.  
2. Получите `peer_id` беседы (см. ниже).

**Если ссылка на беседу такая:**  
`https://vk.com/im/convo/-202321163?...`  

Число после `convo/` — это id беседы. Для API обычно указывают:

```text
VK_NOTIFY_PEER_ID = 2000000000 + 202321163 = 2202321163
```

(берёте **модуль** числа без минуса: `202321163`, прибавляете `2000000000`).

Если отправка не сработает — в [messages.getConversations](https://dev.vk.com/ru/method/messages.getConversations) с ключом сообщества найдите нужную беседу и поле `conversation.peer.id`.

**Вариант Б — личные сообщения админу от сообщества**  
- `VK_NOTIFY_PEER_ID` = ваш числовой **user_id** (например `123456789`).  
- Нужно один раз **разрешить сообщения от сообщества** (напишите в сообщения группы «Начать» / откройте диалог).

**Как узнать chat_id беседы**  
- Временно добавьте в беседу бота или используйте [dev.vk.com](https://dev.vk.com/ru/method/messages.getConversations) с ключом сообщества — в ответе `peer.id` для беседы уже в формате `2000000000 + chat_id`.

## Шаг 2. Cloudflare Worker (бесплатный тариф)

### Всё из терминала (Windows)

```powershell
cd "путь\к\igra\worker"
npm install -g wrangler
.\setup-leads.ps1
```

Скрипт по шагам: login → KV → секреты → deploy → запись URL в `js/config-leads.js`.

---

### Вручную

1. Аккаунт на [cloudflare.com](https://www.cloudflare.com).
2. Установите [Wrangler](https://developers.cloudflare.com/workers/wrangler/install/):
   ```bash
   npm install -g wrangler
   wrangler login
   ```
3. В папке `worker/` проекта:
   ```bash
   cd worker
   wrangler kv namespace create LEADS
   ```
   Скопируйте `id` в `wrangler.toml` → `[[kv_namespaces]]` → `id = "..."`.

4. Секреты (токены не в коде!):
   ```bash
   wrangler secret put VK_GROUP_TOKEN
   wrangler secret put VK_NOTIFY_PEER_ID
   wrangler secret put ADMIN_SECRET
   ```
   - `ADMIN_SECRET` — придумайте длинный пароль для входа в админку (список заявок).

5. Деплой:
   ```bash
   wrangler deploy
   ```
   Получите URL вида: `https://gift-future-leads.ВАШ_АККАУНТ.workers.dev`

6. Проверка:
   - `https://ВАШ_WORKER/health` → `{"ok":true,"vk":true}`

## Шаг 3. Прописать URL в игре

В файле **`js/config-leads.js`** (после деплоя):

```javascript
const LEADS_API = {
  baseUrl: 'https://gift-future-leads.ВАШ_АККАУНТ.workers.dev',
  adminSecret: '',  // пароль только в admin.html, не в GitHub!
};
```

Залейте на GitHub **только baseUrl** (без токена ВК и без adminSecret).

## Шаг 4. Админка

1. Откройте `admin.html`.
2. Вкладка **Заявки** → поле **URL API** и **Ключ админки** (если добавлены в интерфейс) или задайте в `config-leads.js`.
3. **Обновить список** — заявки со всех устройств.

Экспорт JSON по-прежнему работает.

---

## Проверка отправки в ВК

1. Пройдите игру на сайте, отправьте тестовую заявку.
2. В беседе / ЛС должно появиться сообщение с полями заявки.
3. Если ошибка — в консоли браузера (F12) ответ `/lead` с полем `vk.error`; сверьте токен и `peer_id`.

### Частые ошибки ВК

| Ошибка | Решение |
|--------|---------|
| Can't send messages to this peer | Откройте диалог с сообществом или проверьте peer_id беседы |
| Access denied | В ключе нет права «Сообщения сообщества» |
| 901 | Пользователь запретил сообщения — используйте беседу админов |
